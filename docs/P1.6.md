### CHECK 02 — FK columns without index: 🔴 416 findings
Real performance issue. Mỗi FK nên có index để JOIN/CASCADE nhanh.

Sample (auto-fixable DDL có sẵn):
```sql

CREATE INDEX IF NOT EXISTS idx_ai_conversation_tenant_id ON ai.ai_conversation (tenant_id);
CREATE INDEX IF NOT EXISTS idx_api_app_tenant_id ON api.api_app (tenant_id);
-- ... 413 more
```

→ **P1 — apply hàng loạt trong Sprint 2** (1 transaction). Pre-launch không bị slow nhưng load tăng sẽ đau.

### CHECK 03 — RLS gaps: 🔴 8 findings (P0)
**8 tables trong schema `tenant.*` có RLS ENABLED nhưng NO POLICY** — đây là silent bug nguy hiểm: RLS enabled mà không policy = **block toàn bộ SELECT/INSERT/UPDATE/DELETE** từ non-superuser.

Affected tables:
```
tenant.tenant_billing_account
tenant.tenant_branding
tenant.tenant_data_residency_policy
tenant.tenant_domain
tenant.tenant_feature_entitlement
tenant.tenant_overage_charge
tenant.tenant_plan_subscription
tenant.tenant_usage_metering
```

Fix DDL (auto-generated):
```sql
ALTER TABLE tenant.tenant_billing_account ENABLE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON tenant.tenant_billing_account 
  USING (tenant_id = public.current_tenant_id() OR pg_has_role('csr_admin','MEMBER'));
-- (lặp 8 lần cho 8 table)
```

→ **P0 — Sprint 2 priority 1**. Hiện chưa có ai dùng các table này (tenant module mới R22 → chưa wiring) nên không gãy production, nhưng phải fix trước khi bật tenant billing/branding feature.

### CHECK 04 — Audit trigger gaps: ✅ 0 findings
All critical tables (orders, payments, RFQs, users) đều có `trg_*_audit AFTER` trigger.
