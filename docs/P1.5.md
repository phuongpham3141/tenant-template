## Root causes phát hiện

### Cause 1: Middleware thiếu auth gate

```typescript
// VM /app/src/middleware.ts (actual)
import { NextRequest, NextResponse } from "next/server";
import { detectLocaleFromHost } from "@/lib/i18n";

export function middleware(req: NextRequest) {
  const host = req.headers.get("host");
  const locale = detectLocaleFromHost(host);
  const res = NextResponse.next();
  res.cookies.set("locale", locale, { path: "/", sameSite: "lax" });
  return res;
}

export const config = {
  matcher: ["/((?!_next|api|favicon.ico|.*\\..*).*)"],
};
```

**Missing:** Logic check session cookie `csr_session` và redirect tới `/login` cho `/buyer-center/*`, `/seller-center/*`.

### Cause 2: Storefront /api routes không có

```
find /app/src/app/api -type f -name 'route.*'
→ (empty)
```

Không có `/api/auth/register`, `/api/auth/login`, `/api/cart`, `/api/rfqs`. Storefront hiện chỉ là **UI shell** không backend.

### Cause 3: Missing `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` env

```
env | grep MEDUSA
→ NEXT_PUBLIC_MEDUSA_BACKEND_URL=<set>
   (NO NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY)
```

→ Browser không thể gọi `/store/*` (Medusa v2 yêu cầu publishable key header).

**Key đã tồn tại trên Medusa:**
```
pk_61509c15be9863ca278ff99a585d9f663f92b1e507031994c1d73e00360995f9
(title: Default Publishable API Key)
```

Cần inject vào `compose-dev/storefront.env` hoặc `compose-dev/.env`.
