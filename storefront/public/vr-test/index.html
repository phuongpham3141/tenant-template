🧪 VR Fullscreen — 6 phương án test

Mở từng phương án, bấm nút Toàn màn hình, ghi nhận cái nào hoạt động (đúng = phóng to KÍN MÀN HÌNH thật, không phải mở tab Made-in-China). Kết quả mỗi approach in ra ngay trên trang.

  1. Approach 1 — Native Fullscreen API trên wrapper div Cách hiện đang dùng trong Vr360Frame.tsx. Gọi wrapper.requestFullscreen().
  2. Approach 2 — Native Fullscreen API trên iframe trực tiếp Gọi iframe.requestFullscreen(). Mất overlay nhưng giữ iframe content.
  3. Approach 3 — CSS pseudo-fullscreen ONLY Không dùng API native, chỉ position:fixed; inset:0; z-index:9999. Domain giữ nguyên.
  4. Approach 4 — Native API + CSS fallback combined (chi tiết log) Như Approach 1 nhưng có log đầy đủ để xem API có throw lỗi không.
  5. Approach 5 — window.open popup fullscreen Mở cửa sổ mới (vẫn chứa iframe Made-in-China nhưng wrapped bởi page mình).
  6. Approach 6 — Native API trên iframe + allowfullscreen attr Dùng allowfullscreen attribute (cũ) thay vì allow="fullscreen".
📋 Cách báo kết quả: mỗi approach có vùng log màu xanh/đỏ. Chụp màn hình hoặc copy text log gửi lại. Tôi sẽ chọn approach hoạt động và áp vào React component.
